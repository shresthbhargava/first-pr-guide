# Roadmap — first-pr-guide

This document tracks where the project is going. It's public so contributors can see what's planned, pick up something they care about, or suggest something we haven't thought of.

---

## Current Status

`v1.0` — Core guide is live. Covers the full beginner contribution workflow.

---

## Short-Term (Next 4–8 Weeks)

### Content Improvements
- [ ] Add diagrams to `git-basics.md` showing branch flow visually
- [ ] Add a "What is a merge conflict and how do I fix one?" section
- [ ] Add examples of real PRs from the community (with permission)
- [ ] Expand `common-mistakes.md` with real stories from contributors

### New Docs
- [ ] `docs/after-your-first-pr.md` — What to do once you've merged your first PR
- [ ] `docs/understanding-labels.md` — Dedicated deep-dive on GitHub labels
- [ ] `docs/writing-good-issues.md` — How to report bugs and request features properly

### Tooling
- [ ] Add a GitHub Actions workflow to check for broken links automatically
- [ ] Add `all-contributors` bot to recognize all types of contributions

---

## Medium-Term (1–3 Months)

### Video Resources
- [ ] Partner with community members to create short video walkthroughs
- [ ] Embed or link video explanations for complex topics (merge conflicts, rebasing)
- [ ] Add a "Watch instead of reading" note on each page for visual learners

### Community Examples Section
- [ ] Create `examples/` folder with real, annotated PRs
- [ ] Showcase contributions from different backgrounds (docs-only, code, design)
- [ ] Add a "Hall of First PRs" — a place for beginners to list their first merged PR

### Mentorship Section
- [ ] Create `mentorship/` section with:
  - How to find a mentor
  - How to be a good mentee
  - Resources for reverse mentoring (experienced devs learning from newcomers)
  - How to mentor others once you've gotten comfortable

---

## Long-Term (3–6 Months)

### Translation Support
- [ ] Set up `translations/` folder structure
- [ ] Create translation contribution guidelines
- [ ] Priority languages: Hindi, Spanish, Portuguese, Mandarin, Bengali
- [ ] Translation review process — native speakers verify accuracy

### Interactive Learning
- [ ] Build a companion practice repository where beginners can open "safe" PRs
- [ ] Add a self-assessment checklist: "Are you ready for your first real PR?"
- [ ] Create scenario-based examples ("What would you do if...?")

### Program-Specific Guides
- [ ] `docs/programs/gssoc.md` — GSSoC-specific contribution guide
- [ ] `docs/programs/hacktoberfest.md` — Hacktoberfest tips and rules
- [ ] `docs/programs/gsoc.md` — How to position yourself for GSoC

---
